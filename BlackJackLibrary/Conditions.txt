Отлично, теперь у нас есть почти вся нужная логика, для того чтобы приступить к интеграции с UI и получению 
полноценного WPF приложения! Но есть нюанс, консольное приложение не особо вяжется с WPF, плюс они не могут работать 
вместе, т.к. результатом компиляции обоих программ является исполняемая программа с расширением .exe! Сейчас мы это 
исправим и сделаем так, чтобы нашу логику можно было без проблем добавлять в любую программу.
======================================================================================================================


Для этого мы просто изменим тип результата итоговой компиляции нашей программы, для этого:
	1)	Удалите полностью из проекта файл BlackJackLibrary.cs, он нам больше не нужен и весь этот проект он был для вас как некое 
		привычное место для отладки кода.
	2)	Теперь пути назад нету! В Обозревателе решений (Solution Explorer) щёлкните правой кнопкой по названию проекта 
		(в вашем случае это BlackJackLibrary).
	3)	В появившемся контекстном меню найдите нажмите на пункт Свойства (Properties).
	4)	В появившемся окне, во кладке Приложение (Application) (оно кстати будет открыто по умолчанию) найдите выпадающий 
		список Тип выходных данных: (Output type:) и измените значение с "Консольное приложение" (Console Application) 
		на "Библиотека классов" (Class Library).
	5)	Всё, сохраните изменения нажав на CTRL + S или просто кликнув на значок дискеты (забавно, большинство из читающих 
		это застали дискеты именно в виде значков сохранения информации).

Теперь просто пересоберите проект и всё! Если всё сделали, то можно смело отправлять на проверку и наслаждаться дропом + 
переходить ко второй части этого рейда, ведь самое интересное ещё впереди!
